# 点单系统 v1 设计文档

> 文档驱动开发：本文件先于代码，所有实现细节以本文档为准；后续改动先改文档再改代码。

## 1. 目标

做一个轻量级多人点单 H5 系统，主要场景是朋友/同事聚餐时各自扫码点单，最后汇总成一桌的清单。

- 多人点单：通过共享一个 **会话（Session）** 实现，每个人点单都归属到对应人名下。
- 不需要支付：只生成汇总清单。
- 移动端为主，PC 兼容：单列布局 + 自适应宽度，最大宽度 480px 居中。
- 技术栈：Python + Flask，单文件后端，HTML/CSS/JS 不引外部库。
- 数据存储：进程内字典（重启即清空，v1 不接数据库）。

## 2. 角色与流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 1.创建/  │ -> │ 2.进入   │ -> │ 3.选菜品 │ -> │ 4.加入   │
│ 加入会话 │    │ 点单页   │    │ 酱料/附加│    │ 本人购物车│
└──────────┘    └──────────┘    └──────────┘    └──────────┘
                                                       │
                                                       v
                                                ┌──────────┐
                                                │ 5.所有人  │
                                                │ 点完查看  │
                                                │ 汇总清单  │
                                                └──────────┘
```

每个人进入会话后，先填昵称（仅本机会话内有效，用于区分谁点了什么），然后从菜品列表里选，重复操作直到结束；最后任何人可点 "查看汇总"。

## 3. 数据模型

```python
Sauce = {
    "id": "s_001",
    "name": "蒜蓉",
    "price": 0.0          # 酱料默认 0 元；为正数时叠加
}

AddOn = {
    "id": "a_001",
    "name": "加蛋",
    "price": 2.0
}

Dish = {
    "id": "d_001",
    "name": "牛肉面",
    "category": "主食",
    "price": 28.0,
    "description": "手擀面 + 牛腱",
    "sauce_ids": ["s_001", "s_002"],     # 可选酱料
    "addon_ids": ["a_001", "a_002"]      # 可选附加品
}

OrderItem = {
    "id": "uuid",                          # 订单条目 ID
    "session_id": "xxx",
    "user_name": "小王",                   # 点单人昵称
    "user_no": 1,                          # 该会话内的用户序号，详见 §3.1
    "dish_id": "d_001",
    "sauce_ids": ["s_001"],
    "addon_ids": ["a_001"],
    "note": "少辣",                        # 备注，可空
    "unit_price": 30.0,                    # 菜品 + 酱料 + 附加品 计算后单价
    "created_at": "2026-06-03T12:00:00"
}

Session = {
    "id": "6位字母数字",                   # 短码，便于口头/扫码分享
    "title": "周五聚餐",                   # 标题，可空
    "created_at": "...",
    "status": "active" | "closed",
    "user_names": ["小王", "小李", ...],   # 按首次下单顺序记录，序号 = 列表下标 + 1
    "order_ids": [OrderItem.id, ...]
}
```

价格计算：`unit_price = dish.price + sum(sauce.price for s in sauces) + sum(addon.price for a in addons)`。

### 3.1 用户序号规则

- 同一会话内，每个**昵称**在该会话下首次下单时分配一个递增序号（1, 2, 3...）。
- 后续该昵称再下单，序号保持不变。
- 改昵称后再下单 → 视为新用户，分配下一个新序号。
- 序号在汇总页和后台页都展示，格式为 `1号 小王`。

## 4. API 设计

所有响应 JSON，错误统一：`{"error": "msg"}`，HTTP 4xx/5xx。

| Method | Path | 说明 |
| --- | --- | --- |
| GET | `/` | 首页：创建会话 / 输入会话码加入 |
| POST | `/api/session` | 创建会话。body: `{title?}`，返回 `{id, title}` |
| GET | `/api/session/<id>` | 获取会话信息 + 全部订单 |
| POST | `/api/session/<id>/order` | 新增一条订单条目。body: `{user_name, dish_id, sauce_ids, addon_ids, note?}` |
| DELETE | `/api/order/<order_id>` | 删除一条订单条目（仅本人昵称匹配可删，v1 简化为只要会话 active 就能删） |
| GET | `/api/menu` | 返回完整菜单（菜品 + 酱料 + 附加品），用于点单页加载 |
| GET | `/menu/<session_id>` | 点单页 H5 |
| GET | `/summary/<session_id>` | 汇总页 H5（按人分组） |
| GET | `/admin/<session_id>` | 后台汇总页 H5（按菜品聚合） |

会话码生成：6 位 base32 字母数字（去除易混淆字符 0/O/1/I），全局唯一，冲突重试 5 次。

## 5. 页面设计

### 5.1 首页 `/`
- 顶部标题 "点单系统"
- 区块 A：填标题（可选）→ 「创建新会话」按钮
- 区块 B：填会话码 → 「加入会话」按钮
- 样式：居中卡片，最大宽 480px，背景 #f5f5f5

### 5.2 点单页 `/menu/<session_id>`
- 顶部：会话标题 + 会话码（大字可分享） + 「查看汇总」入口 + 「我的昵称」可改
- 菜品分类 tab（如：主食 / 小吃 / 饮品）
- 菜品卡片：名称、价格、描述、加入按钮
- 点 "加入" 弹出底部弹层：
  - 展示当前菜品
  - 酱料多选（单选/多选：v1 用 checkbox 多选，可选 0 个）
  - 附加品多选
  - 备注 textarea
  - "加入我的清单" 按钮 → 弹层关闭
- 底部固定栏：「已点 N 份 · 查看汇总」

### 5.3 汇总页 `/summary/<session_id>`
- 顶部：会话标题
- 按 `user_name` 分组列出所有订单条目，分组标题显示 `序号 + 昵称`（如 `1号 小王`）
- 每条显示：菜品、酱料、附加品、备注、单价
- 底部合计：总金额 + 数量
- 移动端单列；PC 居中，最大宽 720px

### 5.4 后台汇总页 `/admin/<session_id>`
- 顶部：会话标题 + 「按人汇总」入口（跳 `/summary`）
- 按菜品**聚合**：相同菜品的订单合为一组，组内按规格（酱料+附加品+备注）再聚合
- 每行展示：`菜品名 × 份数` + 该规格下 `几号 谁` 点了
- 底部合计：总份数 + 总金额
- 适用场景：服务员/厨房视角，看每样要备几份

## 6. 样式规范

- 主色：`#ff6b35`（橘色，食欲系）
- 辅色：`#2c3e50`（文字）
- 背景：`#f5f5f5`
- 字体：`-apple-system, "PingFang SC", "Microsoft YaHei", sans-serif`
- 圆角：8px
- 移动端：单列、`100vw`、底部安全区预留 `env(safe-area-inset-bottom)`
- PC 端：内容居中，最大宽 480px（点单页）/ 720px（汇总页），背景灰色
